menu_book Que es un Design Doc

Un Design Doc es un documento tecnico que se escribe ANTES de codificar. Explica que se va a construir, como se va a hacer y que alternativas se consideraron. No es una libreria ni un framework — es una practica de ingenieria que los equipos de software mas maduros del mundo usan para reducir errores y alinear al equipo antes de invertir tiempo en codigo.

Analogia: Un Design Doc es al software lo que un plano es a una casa. Puedes construir sin plano, pero vas a tirar paredes. Con un plano, todos saben que se va a construir antes de poner el primer ladrillo.

help_outline Cuando usar Design Docs

No todo necesita un Design Doc. La regla es simple: si la tarea es pequena y directa, ve directo al codigo. Si es grande o toca multiples partes, documenta primero.
