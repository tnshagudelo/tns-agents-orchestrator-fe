settings Dónde se configuran los skills

verified Built-in

Vienen incluidos con Claude Code. No requieren configuración.

/context /clear /compact /commit /fix /simplify
check_circle Disponibles automáticamente al instalar Claude Code
folder Del proyecto

Se guardan en el repositorio. Todo el equipo los comparte vía Git.

.claude/commands/nombre.md
arrow_forward Se invocan con /project:nombre
group Se versionan con el código — todos usan los mismos skills
person Personales

Solo para ti, disponibles en todos tus proyectos.

~/.claude/commands/nombre.md
arrow_forward Se invocan con /user:nombre
laptop Solo existen en tu máquina — no se comparten con el equipo

add_circle Cómo crear un skill

1
Crea el archivo

Crea un archivo .md en la carpeta de commands.

.claude/commands/feature.md
2
Escribe el prompt

El contenido del archivo es el prompt que el agente ejecutará. Usa $ARGUMENTS para recibir parámetros.

description feature.md 
Lee docs/specs/index.md y CLAUDE.md.

Implementa la funcionalidad descrita en: $ARGUMENTS

Sigue las convenciones del proyecto.
Genera tests para los casos principales.
NO modifiques archivos fuera del alcance del spec.
3
Úsalo

Invócalo desde Claude Code pasando los argumentos:

/project:feature docs/specs/notifications.md
tips_and_updates
Tip: comparte los skills con tu equipo

Agrega .claude/commands/ al repositorio. Cuando un nuevo desarrollador clone el proyecto, tendrá los mismos skills disponibles sin configuración adicional.
